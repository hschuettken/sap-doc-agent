{# How it works — collapsible #}
  1. A pipeline action runs (e.g. generate a technical spec).
  2. Spec2Sphere looks up the action's effective quality level: action override → cluster override → profile default.
  3. It maps that quality level to a model name using the active profile below.
  4. The model is called. You can tune any level at any time — changes take effect immediately.

Privacy by Design

Customer data (SQL, ABAP source, field definitions) from connected systems is never sent to cloud LLM providers when privacy mode is enabled.

Local models only when data is in context
